# 遊戲音效專家（Game Audio Specialist）

目標：用一致規範整理全專案音效，確保「好找、好維護、可商用、可快速迭代」。

## 快速路由

- 完整作業流程：`ref/workflow.md`
- 命名規範與資料夾契約：`ref/naming-and-structure.md`
- 外部來源與授權摘要：`ref/sources.md`

## 最小交付（每次音效整理都要做到）

- 列出此次整理範圍（章節、系統、事件、檔案數）。
- 產出/更新一份音效盤點表（含來源、授權、用途、狀態）。
- 定義或套用命名規範，指出未符合規範的檔案。
- 做最少一輪技術驗證：格式、採樣率、峰值、長度、遺失引用。
- 給下一步可執行清單（補音、替換、混音、實作端調整）。

## 標準流程

1. **盤點現況**
   - 掃描現有音效檔，先按用途分類：`ui`, `weapon`, `explosion`, `ambience`, `music`, `voice`, `cinematic`。
   - 建立唯一 ID（例如事件名）並對齊遊戲內觸發點。
2. **授權與來源**
   - 檢查每個素材的來源與 license，標記可商用/需署名/限制用途。
   - 對授權不明素材打 `BLOCKED`，不可進入正式包。
3. **命名與結構**
   - 檔名格式建議：`<type>_<system>_<action>_<variant>_<intensity>_<v###>.wav`
   - 範例：`sfx_ui_button_click_a_soft_v001.wav`
   - 同一事件的變體放同目錄並保留遞增版本。
4. **技術標準**
   - SFX 優先 `wav`, 48kHz, 24-bit（最終包可依平台轉碼）。
   - 保留 headroom，避免硬剪與削波；一致化 loudness 基準。
5. **整合驗證**
   - 對照事件表檢查缺檔、重複檔、未引用檔（orphan）。
   - 針對高頻事件（UI 點擊、射擊、爆炸）做優先 AB 測試。

## 實務規則

- 優先先補「遊戲回饋關鍵音」：命中、受傷、爆炸、UI 成功/失敗。
- 同事件至少 3 個變體，避免聽覺疲勞。
- 音效重整要和關卡節奏一起看，不可只看素材品質。
- 若要上網找新素材，先看 `ref/sources.md` 的可用庫與授權摘要。
