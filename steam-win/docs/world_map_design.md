# 📟 《雷區指令：電報戰線》100 關大戰區總綱 (World Map Design)

## 🎖️ 核心主題設定
* **收報員 (玩家)**：負責解讀由後方發來的摩斯電碼（長官指令）。
* **前線工兵 (小兵)**：負責執行指令，在地圖上精準埋雷。
* **電報系統**：右側 UI 為捲軸式電報紙，數字出現時伴隨 `滴答` 聲。

## 🕹️ 章節核心機制詳解

### 第三章：移動沙塵暴 (Moving Sandstorm)
* **表現**：開局 3 秒全圖可見，隨後一塊 3x3 的雲霧開始沿路徑移動。
* **規則**：被雲霧覆蓋的格子無法看到數字，但可以點擊埋雷。
* **策略**：玩家需在雲霧移走後快速校對，或在雲霧到達前預先佈署。

### 第六章：孤立廢雷 (Ghost Mines)
* **表現**：地圖上預設存在一些灰色、生鏽的地雷。
* **規則**：這些雷**「不參與任何數字計算」**。如果一個格子數字是 3，周圍有一個廢雷和三個玩家佈的雷，則邏輯錯誤（廢雷不計入）。
* **功能**：純粹作為障礙物，增加佈線的難度。

### 第七章：電報輪換 (Telegraph Drift)
* **表現**：手牌區域的數字框會有「跳動動畫」。
* **規則**：數字會根據設定頻率（如每 5 秒）隨機變更為指令池中的另一個數字。
* **策略**：這考驗玩家的「動態規劃」，必須在正確數字出現的窗口期完成操作。

### 第八章：混合地形視覺引導 (Mixed Grid Visuals)
* **理解方案**：
    1.  **區塊染色**：方形區（藍）、三角區（紅）、六角區（黃）。
    2.  **鄰居連線 (Hover Feedback)**：當滑鼠滑過任一格子時，系統會拉出淡淡的螢光線指向所有受此格影響的「鄰居格」。
    3.  **邏輯一致**：不論形狀，數字 $N$ 永遠代表「與我相連的鄰居中有 $N$ 顆雷」。

## 🛠️ 開發實作標記 (For Cursor)
1.  **Grid System**: 需實作 `getNeighbors(cellId)` 函數，支援跨類型索引。
2.  **Timer Component**: 支援 `onTimeUp` 觸發爆炸邏輯。
3.  **Event Dispatcher**: 負責處理 `SandstormMove` 與 `CommandRotate` 事件。
