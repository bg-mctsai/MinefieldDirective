# 📟 《雷區指令：電報戰線》100 關大戰區總綱 (World Map Design)

## 🎖️ 核心主題設定
* **收報員 (玩家)**：負責解讀由後方發來的摩斯電碼（長官指令）。
* **前線工兵 (小兵)**：負責執行指令，在地圖上精準埋雷。
* **電報系統**：右側 UI 為捲軸式電報紙，數字出現時伴隨 `滴答` 聲。

## 🕹️ 章節核心機制詳解

### 第三章：乾谷據點（戰術據點）(Digit Outposts)
* **表現**：地圖上 2～5 個據點格有明顯標示（圖釘＋邊框）；座標由關卡 `levels.json` 的 `digitOutposts` 陣列指定。
* **規則**：每處據點必須出現「數字佈署」（與一般電報埋數相同）；不可把據點僅當成已確認的安全空格而留空。若據點格被邏輯揭示為**地雷**（或動態廢雷佔據該格），**立即失敗**。若總覆蓋率已達關卡目標，但任一想定據點仍無數字，該步結算時**視同失敗並引爆**。
* **策略**：邊推覆蓋邊優先清空據點，避免最後一步「數學上已滿覆蓋」卻因據點未填而翻車。

### 第六章：孤立廢雷 (Ghost Mines)
* **表現**：地圖上預設存在一些灰色、生鏽的地雷。
* **規則**：這些雷**「不參與任何數字計算」**。如果一個格子數字是 3，周圍有一個廢雷和三個玩家佈的雷，則邏輯錯誤（廢雷不計入）。
* **功能**：純粹作為障礙物，增加佈線的難度。

### 第七章：電報輪換 (Telegraph Drift)
* **表現**：手牌區域的數字框會有「跳動動畫」。
* **規則**：數字會根據設定頻率（如每 5 秒）隨機變更為指令池中的另一個數字。
* **策略**：這考驗玩家的「動態規劃」，必須在正確數字出現的窗口期完成操作。

### 第八章：定時炸點 (Blast Point Crisis)
* **表現**：地圖上有 1～5 個特殊格子（炸點），開局即顯示倒數計時（如 `⏱ 18s`）。
* **規則**：
    1.  每個炸點**獨立倒數**，與主關卡計時器無關。
    2.  倒數歸零前，若該炸點周圍格子的地雷邏輯**尚未被完全確認**（即鄰近數字格尚有未解決的雷），立即觸發**爆炸失敗**。
    3.  成功在倒數歸零前確認周圍所有地雷 → 炸點「解除」，可獲主計時器加秒獎勵（`defuseBonusSec`）。
* **難度遞進**：
    * L71～72：1～2 個炸點，倒數 30～35 秒，暖身。
    * L73～75：2～3 個炸點，倒數 22～28 秒，疊加沙塵暴等視覺壓力。
    * L76～78：3～4 個炸點，倒數 18～22 秒，多炸點同時壓制。
    * L79～80：3 個炸點，倒數 15～22 秒，章節 Boss 戰。
* **策略**：玩家必須學會**優先序排程**——倒數最短的炸點先處理，同時兼顧整體覆蓋率目標。

### 第九章：鄰焰共振（熱力迴聲）(Neighbor Placed Digit Bonus)
* **開關**：各關是否在 `levels.json` 設 `neighborPlacedDigitBonus: true`；未設或 `false` 則無此機制（不依章節硬編碼）。
* **表現**：小兵抵達格子上顯示的數字可能高於電報上的底數——周圍每多一格「已佈數字」，最終值 +1。
* **規則**：
    1.  **底數**來自手牌（或信號干擾鎖定值）。
    2.  **鄰接**與 solver 相同：方格為八鄰；三角／蜂巢依 `gridTopology` 的邊鄰接。
    3.  **已佈數字**包含開局 `mapLayout.prePlaced` 轉成的提示格與玩家歷次放置。
* **難度遞進（企劃稿）**：
    * L81～82：方形暖身與三角鑲嵌初體驗；三角關手牌權重僅 1～3（與鄰格上限一致）。
    * L83～86：穿插六角占位與較大方形，疊加沙塵等視覺壓力。
    * L87～90：大方形、更大三角／六角，末關 25×25 方形收束。
* **策略**：先規劃「孤立落點」維持底數，或刻意串聯高鄰數格換取高約束——兩種路線都需心算加成。

## 🛠️ 開發實作標記 (For Cursor)
1.  **Grid System**: 需實作 `getNeighbors(cellId)` 函數，支援跨類型索引。
2.  **Timer Component**: 支援 `onTimeUp` 觸發爆炸邏輯。
3.  **Event Dispatcher**: 預留擴充（`levels.json` 之 `events` 尚未實裝）；`mapCloudOverlay` 沙幕若啟用僅為視覺層，與據點機制獨立。
4.  **Blast Point System** (第八章)：
    - `blastPoints` 陣列從 `LevelDefinition` 讀入，每個炸點在畫面上顯示獨立倒數 UI（格子覆蓋層，含動畫）。
    - 倒數結束時呼叫 `checkBlastPointSatisfied(pos)` 判斷鄰近是否完全確認；未滿足則觸發 `onBlastExplode` → 強制失敗。
    - 成功解除時呼叫 `addMainTimerBonus(defuseBonusSec)` 加秒。
5.  **鄰焰共振／熱力迴聲** (第九章)：
    - 關卡 JSON 設 `neighborPlacedDigitBonus: true`。
    - `useMineGame` 在佈署前以 `logicNeighborKeys` 計算目標格相鄰「已佈數字」格數，加到電報底數後再送入 `MineSolver`。
6.  **戰術據點** (第三章等，由 JSON 開關)：
    - `digitOutposts: [[x,y],…]` 從 `LevelDefinition` 讀入；達覆蓋率時逐一檢查 `placedNumbers` 是否含該座標，否則 `onOutpostIncomplete` → 與邏輯爆相同的連鎖失敗流程。
