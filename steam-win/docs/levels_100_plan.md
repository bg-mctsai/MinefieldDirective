# 🗺️ 100 關全戰役規劃書 (100-Level Strategic Campaign)

本文件與 [world_map_design.md](./world_map_design.md) 的**十大戰區**一致：**每 10 關為一章**（`Lv.1–10` … `Lv.91–100`），共 100 關。

## 0. 核心邏輯調整
* **數字難度係數**：
    * **簡單 (Level C)**：`3`, `4`, `5`, `6` (周圍空間大，容錯高)
    * **中等 (Level B)**：`2`, `7` (邏輯點變窄)
    * **困難 (Level A)**：`1`, `8` (極度精確，1 代表周圍只能有一顆雷，8 代表必須全滿)
* **過關條件**：
    * **覆蓋率 (Coverage)**：必須正確填滿地圖指定比例的格子。
    * **邏輯一致性**：所有已填數字必須完全符合邏輯，不可爆炸。

## 0.1 關卡挑戰與解鎖規則
* **關卡總數固定**：`1..100`（共 100 關）。
* **章節對應**：`chapter = ceil(level / 10)`，範圍 `1..10`（與戰區編號一致）。
* **初始解鎖**：新存檔只解鎖 `Level 1`；`Level 2..100` 預設鎖定。
* **線性解鎖**：通關 `Level N` 後更新進度 `highestClearedLevel = max(highestClearedLevel, N)`，並解鎖 `Level (highestClearedLevel + 1)`（若 `<= 100`）。
* **不可跳關**：UI 與進入戰鬥流程都必須檢查解鎖條件；未解鎖關卡不可進入戰鬥。
* **存檔防呆**：讀取進度若異常，需 `clamp` 到 `0..100`。

## 2. 與 world_map 的機制對照

以下細節以 [world_map_design.md](./world_map_design.md) 為準，本表僅標註**首次出現章節**：

* **第三章（乾谷據點）**：`digitOutposts` 據點格必須佈數字；據點不得為邏輯雷／廢雷；覆蓋率與據點雙條件。
* **第六章（深海要塞）**：孤立廢雷不計入鄰居雷數。
* **第七章（信號干擾區）**：電報輪換頻率與動態規劃。
* **第八章（維度裂縫）**：區塊染色、Hover 鄰居連線、混合 `getNeighbors`。

**角色／劇情解鎖**（若與戰役綁定）：建議在 **章 3～4** 引入工程師類職能、**章 4～5** 強化三角／六角專精角色、**章 7～8** 情報／干擾對應角色——實作時以 `levels.json` 或劇情表另行定稿，與上表不衝突即可。

## 3. 數據結構參考 (JSON 結構)

為了讓 Cursor 幫你生成關卡，建議定義如下格式（`chapter` 必須與 `ceil(level/10)` 一致）：

```json
{
  "level": 25,
  "chapter": 3,
  "mapType": "RING_OR_FRAGMENT",
  "gridType": "RECTANGLE",
  "config": {
    "size": { "x": 10, "y": 10 },
    "targetCoverage": 0.75,
    "timeLimit": 90,
    "allowedInstructions": [1, 2, 3, 4, 5, 6, 7, 8],
    "specialEvent": "SANDSTORM_MOVE"
  }
}
```

`specialEvent` 建議枚舉與 world 對齊，例如：`NONE`、`TIMER`、`SANDSTORM_MOVE`、`GHOST_MINES`、`TELEGRAPH_DRIFT`、`MIXED_GRID`、`CHAIN_OVERHEAT`、`FINAL_COLLAPSE`（終章複合事件）。
